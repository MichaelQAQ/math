编程创作
成长轨迹

编程不是只学语法，而是学习怎样把想法变成可以运行、可以修改、可以分享的作品。

查看成长线 当前方向

像搭建一间会思考的工作室。

从第一个脚本开始，把重复劳动交给程序，把抽象逻辑变成界面，把学习过程留下可继续生长的痕迹。

成长线

编程学习的重点，是从“照着写”慢慢走到“自己设计”。

01语法感

先把代码读顺，再把小任务写出来

理解变量、条件、循环和函数，让代码从陌生符号变成清晰表达。

  • 用 Python 完成计算、整理和自动化小任务。
  • 把常见错误写成调试笔记。
  • 训练“先拆步骤，再写代码”的习惯。
02工具感

让程序真正帮自己学习和工作

工具不是炫技，而是让一个好想法有稳定的落地方式。

  • 制作学习计划、卡片、资料整理和网页工具。
  • 学习文件、数据、浏览器和接口的基本操作。
  • 把常用流程做成可重复使用的小组件。
03网页

把作品放到浏览器里，让别人也能进入

HTML、CSS、JavaScript 是表达作品的舞台，也是学习设计和交互的入口。

  • 练习清晰布局、按钮状态和移动端适配。
  • 做数学演示、学习记录和作品展览页。
  • 用 GitHub Pages 学会发布与迭代。
04算法

从会写程序走向会设计方法

算法训练的是耐心、结构感和对复杂问题的分解能力。

  • 理解排序、搜索、递归和动态过程。
  • 用可视化方式观察算法运行。
  • 把算法和数学直观连接起来。
05创作

把代码变成自己的作品档案

每个小项目都是一次表达：说明问题、设计方案、做出界面、不断修正。

  • 积累网页、动画、小应用和教学工具。
  • 给作品写说明、截图和版本记录。
  • 用代码把想象力变成可体验的东西。
